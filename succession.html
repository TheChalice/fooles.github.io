Nodejs

Nodejs 入门

JS是脚本语言，脚本语言都需要一个解析器才能运行。对于写在HTML页面里的JS， 浏览器充当了解析器的角色。而对于需要独立运行的JS，NodeJS就是一个解析器。 每一种解析器都是一个运行环境，不但允许JS定义各种数据结构，进行各种计算, 还允许JS使用运行环境提供的内置对象和方法做一些事情。例如运行在浏览器中的JS的 用途是操作DOM，浏览器就提供了document之类的内置对象。而运行在NodeJS中的JS的 用途是操作磁盘文件或搭建HTTP服务器，NodeJS就相应提供了fs、http等内置对象。@more

通过圣斗士星矢理解继承

  • 星矢不穿圣衣：能力一般

  • 穿上青铜圣衣：变强

  • 穿上黄金圣衣：拥有了黄金圣衣带来的力量以及黄金一箭

  • 女神的能量：摸手

  • 领悟到第七感：变得更厉害

  • 总结：星矢通过穿上黄金圣衣拥有了黄金圣衣的力量和黄金箭
    这就是我们今天要讲的继承。

从现实的角度理解继承

儿子女儿和父亲母亲长得很像包含两层含义：
儿子女儿拥有了父亲母亲的某些特性（方法属性）比如：双眼皮，单眼皮，身高，柳叶眉，刀眉，剑眉，聪明（主要是继承在母亲）.
同时，儿子，女儿还有自己的一些特性，也就是儿子，女儿并不是和父亲或者母亲长得一模一样
从父母 儿子的角度理解继承。从两点分析：

  • 第一点：我们长得和我们父母很像
    眉毛 鼻子 嘴巴 身高 天赋

  • 第二点：我们还有自己的一些特点
    我们的某些部位可能和父母不一样(前提你的邻居不姓王)；

从商城中分析继承

思考普通类产品和服务类产品的相同点，不同点：
普通产品：书籍 手机 电脑
服务类产品：旅游 充值 团购（聚餐） 电影票
分析不同点和相同点
相同点：
名称 图片 描述 价格 购买
属性的不同点：
衣服：尺寸XX XLL XLLLLL 款式 男款 裙子 设计师
电子书：版权 出版社 作者 ISBN 出版号 出版日期
方法的不同点（购买流程）：

  • 书籍购买：点击购买--生成订单--物流 --收货

  • 电影票购买：点击购买--生成订单--托管金钱--生成一个二维码--购买者拿着这个二维码或者凭证线下去电影院去让人家扫一下--这个时候平台才把金钱打到电影院的银行卡里面--完成这个流程

  • 电子书购买：试读--购买之后允许阅读全部内容
    总结 ：名称都是购买，但是购买的流程不一样，就是代码不一样
    Buy（）--抽象的
    具体的购买是根据每个产品的不同而不同的
    都叫buy，但是实现的代码是不一样的

第一层 Object中的继承 – 最简单的继承—万物都继承在object

万物都继承在object
准确的说：万物都继承自object.prototype
也就是说 所有的对象不管是内置，还是自定义都是继承自object
两个含义：

  • 第一层：拥有了object 的所有特性（属性 方法）

  • 第二层：我还拥有自己的特性
    toString（）
    Array String 等对象一般会重写toString（）
    Object.prototype.toString()
    比如：String对象他就拥有tostring方法还拥有自己的方法：比如substr，substring。。。。。也就是：js中的一切对象都能够访问object中的方法

第二层 内置对象的继承关系

所有的内置对象都是其首字母大写的类型的构造函数实例出来的
所以所有实例原型都指向其构造函数所有构造函数的原型对象都指向Function.prototy.
Function.prototy原型对象是Object.prototype
