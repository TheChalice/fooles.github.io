设计模式(design model)

设计模式是可重用的用于解决软件设计中一般问题的方案。设计模式如此让人着迷，以至在任何编程语言中都有 对其进行的探索。 其中一个原因是它可以让我们站在巨人的肩膀上，获得前人所有的经验，保证我们以优雅的方式组织我们的代码，满足我们解决问题所需要的条件。 设计模式同样也为我们描述问题提供了通用的词汇。 这比我们通过代码来向别人传达语法和语义性的描述更为方便。@more

Node.js介绍

  • 客户端和服务器端

    • 面向用户的终端设备，前端说白了就是在给各种终端设备开发界面交互（安卓（Java）、IOS（C语言、Object-C）都是前端）

    • 我们的前端：大前端，（全栈工程师），使用Web技术来为各种平台构建应用解决方案

    • 要想让我们的数据动起来，需要一个后台，说白了就是服务器端开发

    • Node.js是一个可以把静态网页给动态化的一种技术

  • 什么是Node.js？

    • Node.js是一个 在浏览器之外，可以解析和执行JavaScript代码的运行环境

  • Node.js基于哪个JavaScript引擎？

    • 基于Chrome的 V8 引擎而构建的

  • Node.js可以做什么？

    • 开发网站

    • 只要是网站，Node.js就可以做

    • 但是Node.js有自己的特殊领域，针对某一方面的服务应用有特别好的优势

      • Node.js特别适合处理具有高并发，但是对CPU消耗一般的应用

      • 很多人 同时来请求访问我们的服务器

安装和配置Node.js开发环境

path环境变量

环境变量的本质就是用来让你可以快速的定位到一个软件的目录

当我们在命令台中输入一个程序的名称的时候，命令台默认把它当成是当前目录下的一个可执行文件来打开，
如果找不到：
去系统的path环境变量中找，一个路径一个路径的挨着尝试查找该可执行文件
如果path环境变量找也找不到：
直接报错：'feiq' 不是内部或外部命令，也不是可运行的程序或批处理文件。

  • 打开系统属性

    • 在开始中输入sysdm.cpl敲回车就可以了

    • 右键我的电脑 - 属性 - 高级系统设置 - 切换到”高级“tab栏 - 选择 环境变量

  • 如何使用

    • 可以直接在path里面添加可执行文件所处的目录，但是不建议这么做，不好修改，太乱了

    • 在外部定义一个变量

      • 变量名的规范就是：程序的名称_HOME

      • 变量值：就是程序所处的目录

    • 在path中通过 ;%变量名%; 的格式引用外部的一个变量

cmd（Command）

作用：这是一个终端，可以在里面把以前鼠标点点点的方式通过命令来让计算机帮你做一些事情

  • cmd打开的方式

    • 第一种通过：win+r 输入cmd 敲回车就可以进入cmd控制台了

    • 通过开始- 程序 - 附件 找到 命令提示符

    • cd （ change directory）改变和切换目录的位置

      • cd ../ 切换到上一级目录

      • cd 目录名 切换到到指定的目录

    • dir（directory）表示列出当前目录下一级所有子文件或子目录

    • md（make directory）

    • rd（remove directory）

    • del（delete）

    • cls （clear）

nvm

Node version Manager 这是一个Node.js的版本管理工具

  • nvm install 版本号

  • nvm uninstall 版本号

  • nvm use 版本号

  • nvm list|ls 查看当前已安装的所有的Node.js版本

简单使用

  • 在控制台通过输入node命令来执行一个js脚本文件

    
      node example.js 敲回车

  • 也可以直接省略扩展名

REPL运行环境

Read Eval Print Loop

REPL 可交互式运行环境，和浏览器中的Console是一样的

作用：可以帮助我们测试JavaScript代码，或者是一些Node.js的API

  • 按住Ctrl 再按 c 两次，就可以退出REPL运行环境

  • 在REPL环境中输入 .exit 也可以退出REPL环境
